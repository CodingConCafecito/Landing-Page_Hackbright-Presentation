Meaningful Takeaways

Pseudocode

  • Writing It
    • Don't worry too much about how you will organize the code into different functions, you will figure that out later. Get down as much of your thinking and logic as possible.
    • Have very clear distinctions on what are requirements versus nice to haves. You can modify these lists later as you program, but get one down at the start to have a clear plan of action.

Programming:

  • Commenting
    • It's super helpful to comment as you code, especially when you learned something new. It makes it so much easier to pick up where you left off.
    • Writing out the logic in non-technical terms also helps you communicate your code others.

Debugging

  • Testing
    • Run your code constantly, especially early on and when you are unsure. This will help you save time later.
  • GitHub Desktop
    • Review changes before committing to see how you've progressed with that bug.
  • PythonTutor.com
    • Don't guess! Use this resource to visually see what is going on.
    • Googling is helpful, but if you don't understand someone else's logic, run their code through this visualizer. You'll learn so much making sense of it for yourself.
